# Notification System

Stay informed with real-time email notifications and in-app alerts for case updates, assignments, deadlines, and important system events.

## Overview

Disclosurely's notification system ensures team members and reporters never miss critical updates. Configure personalized notification preferences, manage alert frequency, and receive timely updates via email and in-app notifications.

**Access:** Dashboard > Settings > Notifications

## Notification Types

### For Case Handlers & Administrators

**Case Assignments:**
- New case assigned to you
- Case reassigned to different handler
- Case escalated to your attention
- Bulk assignments completed

**Status Updates:**
- Case status changed (any case you're involved in)
- Case priority elevated or reduced
- Case marked as resolved
- Case reopened after closure

**Messaging:**
- New message from reporter
- Reporter responded to your question
- Internal note added by team member
- File uploaded by reporter

**Deadlines & SLAs:**
- SLA deadline approaching (24h warning)
- SLA deadline breached
- Response time exceeding target
- Investigation milestone approaching

**System Alerts:**
- New high-priority case requires immediate attention
- Pattern detected across multiple cases
- Security alert or unusual activity
- System maintenance scheduled

### For Reporters

**Submission Confirmation:**
- Report successfully submitted
- Tracking ID provided (DIS-XXXXXXXX)
- Expected response timeline
- Next steps information

**Status Updates:**
- Investigation status changed
- Case assigned to investigator
- Additional information requested
- Investigation completed
- Resolution communicated

**Messaging:**
- New message from investigator
- Question requires your response
- Investigation update available
- Case closed notification

**Reminders:**
- Pending messages requiring response
- Draft expiring soon (if applicable)
- Follow-up requested

## Managing Notification Preferences

### Email Notifications

**Configure Email Settings:**

1. Navigate to Dashboard > Settings > Notifications
2. Select notification categories to enable/disable
3. Choose email frequency (instant, daily digest, weekly summary)
4. Set quiet hours (no emails during specified times)
5. Save preferences

**Email Frequency Options:**
- **Instant:** Email sent immediately when event occurs
- **Hourly Digest:** Batched updates sent once per hour
- **Daily Digest:** Single email at chosen time (e.g., 9 AM)
- **Weekly Summary:** Weekly roundup of all activity
- **Disabled:** No email notifications

**Recommended Settings:**
- High-priority cases: Instant
- Status changes: Daily digest
- Team updates: Weekly summary
- System alerts: Instant

### In-App Notifications

**Real-Time Alerts:**

Notifications appear in the application:

**Notification Bell:**
- Icon in top navigation bar
- Badge shows unread count
- Click to view notification center
- Mark individual or all as read

**Notification Center:**
- Chronological list of all notifications
- Filter by type or date
- Quick actions (view case, reply, dismiss)
- Search notifications

**Desktop Notifications:**
- Browser push notifications (opt-in)
- Appear even when tab not active
- Click to navigate directly to case
- Requires browser permission
